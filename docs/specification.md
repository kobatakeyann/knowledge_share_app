# 社内ナレッジ循環アプリ（デモ）仕様書

## ■ 概要

本アプリは、社内の知見共有および社員同士の交流を促進することを目的としたWebアプリである。
従来の掲示板やチャットツールでは活性化しにくかった「雑な発信」「心理的ハードルの低い交流」を中心に設計する。

### コンセプト

- 雑な投稿から価値あるナレッジを生み出す
- 人と知見が自然につながる
- 貢献が見えることで継続利用を促す

---

## ■ 機能仕様

---

# ① 投稿機能（コア）

## 投稿タイプ

```ts
type PostType = "knowledge" | "question" | "casual";
```

- knowledge：ナレッジ共有
- question：質問
- casual：つぶやき・悩み（メイン）

---

## 投稿データ構造

```ts
type Post = {
	id: string;
	type: PostType;
	content: string;
	tags: string[];
	mood?: "troubled" | "thinking" | "sharing";
	reactions: Reaction[];
	authorId: string;
	createdAt: string;
};
```

---

## mood（温度感）

- troubled（困っている）
- thinking（相談したい）
- sharing（軽い共有）

---

## UI仕様（バルーン）

- サイズ：リアクション数に応じて変化
- 色：
  - knowledge → 青
  - question → 緑
  - casual → ピンク系

- ホバーで詳細表示

---

# ② リアクション機能

## 種類

```ts
type ReactionType =
	| "agree" // わかる
	| "trouble" // それ困る
	| "experience" // 自分もやった
	| "later"; // 後で見る
```

---

## Reaction構造

```ts
type Reaction = {
	id: string;
	type: ReactionType;
	userId: string;
};
```

---

## 挙動

- クリックで即時反映
- 合計数でバルーンサイズ変更
- 一人1リアクション/種別まで

---

# ③ ナレッジ昇格機能

## 条件

- リアクション数が一定以上（例：5以上）
- またはコメント数（MVPでは省略可）

---

## 機能

- 「ナレッジ化」ボタン表示
- 押下で type を knowledge に変更
- タグの再設定を促す

---

# ④ タグ機能

## タグカテゴリ

```ts
type TagCategory = "tech" | "process" | "industry";
```

---

## 例

- tech：React / Java / AWS
- process：要件定義 / 設計 / テスト
- industry：金融 / 通信

---

## UI

- 投稿時：最大3つまで選択
- クリックでフィルタリング
- タグ→有識者導線あり

---

# ⑤ スキル・ユーザー機能（②）

## ユーザー構造

```ts
type User = {
	id: string;
	name: string;
	skills: string[];
	contributions: number;
	recentActivities: number;
};
```

---

## 表示内容

- スキルタグ
- 回答数
- リアクション獲得数
- 最近の活動量

---

## 機能

- タグ検索 → ユーザー一覧表示
- 「この人に相談」ボタン（ダミーでOK）
- 投稿からユーザーへ遷移可能

---

# ⑥ ダッシュボード機能（⑥）

---

## ① 今日のトピック

- ランダム or 人気投稿表示
- 「最近解決された問題」

---

## ② 感謝ログ

```ts
type Thanks = {
	fromUserId: string;
	toUserId: string;
	message: string;
};
```

- 「〇〇さんがあなたに感謝しました」

---

## ③ 貢献ヒートマップ（簡易）

- タグごとの投稿数・リアクション数
- 棒グラフ or バッジ表示

---

## ④ レコメンド投稿

- 自分のスキルタグに一致する投稿
- 未解決投稿を優先表示

---

# ⑦ フィルター機能

- 投稿タイプ別
- タグ別
- 未解決のみ

---

# ⑧ 状態管理（MVP）

```ts
type AppState = {
	posts: Post[];
	users: User[];
	currentUser: User;
};
```

---

# ⑨ UI/UX要件

## 重視ポイント

- 投稿ハードルを下げる
- 一目で理解できる
- 業務中でも使いやすい

---

## 注意点

- アニメーションは軽量にする
- 初期はシンプルなリストUIでも可
- バルーンは装飾レベルからスタート

---

# ⑩ MVP優先順位

## 必須

- 投稿（3種別）
- タグ
- リアクション
- フィード表示

## 次点

- ユーザー一覧
- ダッシュボード

## 余裕あれば

- ナレッジ昇格
- レコメンド

---

# ■ 期待されるユーザー体験

1. 気軽につぶやく
2. 共感・リアクションがつく
3. 有識者が見つけて反応
4. 解決される
5. ナレッジとして蓄積
6. 貢献が可視化される

---

# ■ 補足

このデモではバックエンドは不要とし、すべてフロントで状態管理する。
UI/UXの体験を重視すること。
